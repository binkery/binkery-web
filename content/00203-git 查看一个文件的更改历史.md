<!--markdown-->使用Git 跟踪单个文件的修改历史。


<!--more-->


开发的时候经常会碰见这样的情况，需要跟踪单个文件的修改历史，在CVS 或者 SVN ， 有一些GUI工具帮助做这些工作，在git上应该也有，不过一直没有发现。可以通过命令行来查看修改历史，也还算比较方便。

查看详细历史

    git log -p filename

查看提交记录

    git log filename

记录下需要恢复的 commit 版本号

恢复该文件：

    git reset commit 版本号 filename

提交git:

    git commit -m "注释"
